Prevent Empty Service Account
CIS Kubernetes Benchmark 5.1.5
Ensures that a service account must be explicitly specified in pod specifications. The default service account should not be used implicitly by omitting the serviceAccountName field.

Rule: prevent-empty-service-account

Requires pods to explicitly specify a serviceAccountName field

YAML Configuration
- name: prevent-empty-service-account
  match:
    any:
    - resources:
        kinds:
        - Pod
  validate:
    message: >-
      CIS-5.1.5: A service account must be explicitly specified. 
      The default service account should not be used implicitly.
      Please specify a dedicated service account for your application.
    pattern:
      spec:
        serviceAccountName: "?*"
